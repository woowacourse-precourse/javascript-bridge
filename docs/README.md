# 돌다리도 두들겨보고 건너기 🪨 🪨 🪨

## 📍 **미션 내용**
- 다리 건너기 게임을 구현해보자

<br>
<hr/>

## 📍 **입력 요구 사항**

  - 자동으로 생성할 다리의 길이를 입력 받음(3이상 20이하의 숫자)
  - 라운드마다 플레이어가 입력할 칸을 입력 받음(U - 위 칸, D - 아래 칸)
  - 게임 재시작 / 종료 여부를 입력 받음(R - 재시작, Q - 종료)

<br>
<hr/>


## 📍 **출력 사항 상세**

|상황|출력 문구|비고
|:---:|:---:|:---:|
게임 시작 문구 | 다리 건너기 게임을 시작합니다. | -
게임 종료 문구 - 1 | 최종 게임 결과 | -
게임 종료 문구 - 2 | 게임 성공 여부 : 성공 | "성공" 혹은 "실패"로 출력
게임 종료 문구 - 3 | 총 시도한 횟수 : N | -
다리 길이 입력 | 다리의 길이를 입력해주세요. | 3이상 20 이하의 숫자가 아닐시 예외처리
이동 칸 선택 | 이동할 칸을 선택해주세요. (위: U, 아래: D) | "U" 또는 "D"가 아닐시 예외처리
게임 재시도 여부 | 게임을 다시 시도할지 여부를 입력해주세요. (재시도: R, 종료: Q) | "R" 또는 "Q"가 아닐시 예외처리
현재 다리의 상황 | [ O l X l X ] <br/> [ O l X l X ] | 출력값 주의 <br/> 2줄로 위 아래 표현

## 📍 **필요 기능**

- ### <b>App</b> [게임 - 시작]
  1. 앱을 시작하여 게임을 시작하는 메서드 - play()
  2. 1회 게임의 이동 관련 로직이 포함된 메서드 - game()

<br/>

- ### <b>BridgeGame</b> [게임 - 액션]
  1. 사용자가 칸을 이동하도록 하는 메서드 - move()
  2. 사용자가 다시 게임을 시도하도록 하는 메서드 - retry()
  3. 올바른 칸인지 확인하여 성공 실패 여부를 알려주는 메서드 - cross()

<br/>

- ### <b>BridgeMaker</b> [게임 - 조건]
  1. 다리를 만들어 반환해주는 메서드 - makeBridge(size,generateRandomNumber)

<br/>

- ### <b>BridgeRandomNumberGenerator</b> [Helper]
  1. 0과 1 숫자를 무작위로 생성해 반환해주는 매서드 - generate()

<br/>

- ### <b>InputView</b> [입력]
  1. 브릿지 숫자를 입력받는 매서드 - readBridgeSize()
  2. 사용자가 이동할 칸을 입력받는 매서드 - readMoving()
  3. 사용자가 게임을 다시 시도할지 종료할지 여부를 입력받는 매서드 - readGameCommand()

<br/>

- ### <b>OutputView</b> [출력]
  1. 현재까지 이동한 다리 상태를 출력하는 메서드 - printMap()
  2. 게임의 최종 결과를 양식에 맞게 출력하는 메서드 - printResult()


<br/>

## 📍 **요구 사항**
- ### 1. indent의 depth는 2 이하로 구현
- ### 2. 함수는 한 가지의 일만 하도록 구현
- ### 3. Jest를 통해 기능 테스트 진행
- ### 4. 함수(또는 메서드)의 길이는 10라인 이하
- ### 5. 메서드의 파라미터는 최대 3개까지만 허용
- ### 6. else를 지양
- ### 7. 도메인 로직에 단위 테스트 구현

<br>

## 📍 **기능 별 요구사항**

- ### <b>InputView</b>
  - #### 객체를 활용해 구현 
  - #### 파일 경로는 변경할 수 있다
  - #### 메서드의 인자는 변경할 수 있다
  - #### 사용자 값 입력을 위해 필요한 메서드를 추가할 수 있다
  - #### InputView에서만 MissionUtils의 Console.readLine()을 이용해 입력을 받을 수 있다.

- ### <b>OutputView</b>
  - #### 객체를 활용해 구현 
  - #### 파일 경로는 변경할 수 있다
  - #### 메서드의 이름은 변경할 수 없고, 인자는 필요에 따라 추가하거나 변경할 수 있다
  - #### 값 출력을 위해 필요한 메서드를 추가할 수 있다.

- ### <b>BridgeGame</b>
  - #### 클래스를 활용해 구현
  - #### 필드(인스턴스 변수)를 추가할 수 있다
  - #### 파일 경로는 변경할 수 있다
  - #### 메서드의 이름은 변경할 수 없고, 인자는 필요에 따라 추가하거나 변경할 수 있다
  - #### 필요한 메서드를 추가 하거나 변경할 수 있다
  - #### InputView, OutputView를 사용하지 않는다

- ### <b>BridgeMaker</b>
  - #### 객체를 활용해 구현해야 한다.
  - #### 프로퍼티를 추가할 수 없다
  - #### 파일 경로는 변경할 수 없다
  - #### 메서드의 시그니처(인자, 이름)와 반환 타입은 변경할 수 없다

- ### <b>BridgeRandomNumberGenerator</b>
  - #### Random 값 추출은 제공된 BridgeRandomNumberGenerator의 generate()를 활용
  - #### 코드는 변경할 수 없다.
  - #### 다리 칸을 생성하기 위한 Random 값은 다음과 같이 추출한다. 
    - #### const number = generateRandomNumber();
